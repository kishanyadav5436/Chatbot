ABSTRACT

The Inclusivity AI Chatbot is an innovative web-based application engineered to transform Diversity, Equity, and Inclusion (DEI) education through conversational AI. Traditional DEI resources often face significant hurdles, including limited accessibility, a lack of user interactivity, and substantial language barriers. This project addresses these challenges by delivering a multilingual platform supporting 12 languages, with a specific focus on major Indian languages to ensure localized impact.
The system is built on a sophisticated hybrid AI architecture that integrates traditional Natural Language Processing (NLP) with modern generative models. By combining SpaCy and Scikit-learn for intent classification, Sentence Transformers for semantic search, and the Google Gemini LLM for context-aware responses, the chatbot provides highly accurate and human-like interactions. The technical stack features a Flask backend, MongoDB for conversation history management, and secure user authentication via JWT and Google OAuth. To enhance accessibility, the system incorporates a Voice Interface using the Web Speech API.
Empirical testing demonstrates the system's robustness, achieving an intent classification accuracy exceeding 85% at a 50% confidence threshold and maintaining a low average response latency of less than 2 seconds. This project represents a practical application of advanced AI/ML techniques for social good, offering a scalable solution for inclusive education and global awareness.
Keywords: DEI Chatbot, Multilingual NLP, Hybrid AI, Flask, Google Gemini, Voice Interface, Semantic Search.

CHAPTER 1: INTRODUCTION

1.1 BACKGROUND OF STUDY
Diversity, Equity, and Inclusion (DEI) initiatives are crucial in modern society, particularly in diverse nations like India. Traditional DEI training methods rely on static documents and videos, lacking interactivity and personalization. Language barriers further limit access in non-English speaking regions.

This project introduces Inclusivity AI Chatbot, a conversational AI system that makes DEI education accessible, interactive, and multilingual.

- Lack of interactive learning experiences

3.3 ALGORITHMS USED

ALGORITHM 1: INTENT CLASSIFICATION
Input: User query Q
1. Preprocess Q -> tokens
2. TF-IDF vectorization V(Q)
3. Predict: class = SGDClassifier.predict(V(Q))
4. Confidence = max(prob)
Output: Intent label, confidence

ALGORITHM 2: SEMANTIC SEARCH
Input: Query embedding E(Q), Dataset embeddings E(D)
1. Compute similarities S = cosine(E(Q), E(D_i)) for all i
2. Retrieve top-K with S > threshold
Output: Relevant DEI responses

ALGORITHM 3: LLM FALLBACK
Input: Query Q, context C
Prompt = f"DEI Expert: {Q} Context: {C}"
Response = Gemini.generate(Prompt)
Output: Generated response

CHAPTER 4: CODING, TESTING AND RESULT ANALYSIS

4.1 STEP-BY-STEP IMPLEMENTATION PROCESS (ACTUAL CODE-BASED)

STEP 1: BACKEND SETUP - api_server.py (Flask App)
Created main Flask server with CORS, MongoDB (pymongo), JWT auth
Key: @token_required decorator, pymongo client[d:/Chatbot/backend/inclusivity-chatbot]

STEP 2: AUTHENTICATION IMPLEMENTATION
- JWT encode/decode with HS256
- bcrypt.hashpw for passwords
- Google OAuth: authlib, server_metadata_url with verify_iss=False fix
Endpoints: /api/auth/register, /login, /guest, /google/login+callback

STEP 3: NLP INTENT CLASSIFICATION - nlp_service.py
- Pipeline: TfidfVectorizer(spacy_tokenizer) + SGDClassifier(loss='log_loss')
- Training: YAML nlu.yml -> fit(texts, labels), predict_proba for confidence >0.5
- 25+ intents: greet, goodbye, ask_bias, ask_diversity, nlu_fallback
Code: intent_classifier = IntentClassifier(); .train()

STEP 4: LLM SERVICE - llm_service.py (Hybrid Fallback)
- google.generativeai.GenerativeModel('gemini-pro')
- get_bot_response(intent, message) -> ML responses dict or LLM
- Context-aware prompts: "DEI Expert: {query}"

STEP 5: CHAT API - /api/chat POST
1. intent = intent_classifier.predict(message)
2. conversation_tracker.add_message()
3. bot_reply = get_bot_response()
4. Mongo save: users.conversations.push(messages)
Concurrent safe with ThreadPoolExecutor

STEP 6: FRONTEND - Frontend/index.html + script.js
- TailwindCSS responsive UI with dark/light theme
- Voice: Web Speech API SpeechRecognition()
- Chat queue: Promise.all(messages.map(process))
- History: /api/chat/history GET -> sidebar

STEP 7: DATA LOADING - data_loader.py
- DEI Dataset.csv (5000+ entries), nlu.yml -> MongoDB dei_data
- Precompute embeddings for semantic search

4.2 DETAILED MODULES FROM ACTUAL CODE

MODULE 1: MAIN API (backend/api_server.py - 500+ lines)
- Routes: /api/chat, /history, /auth/*
- token_required(@wraps): jwt.decode -> user_id, is_guest check
- get_bot_response(): ML dict fallback to Gemini

MODULE 2: NLP (backend/nlp_service.py)
def predict(text): TF-IDF -> probas -> argmax, conf>=0.5 else 'nlu_fallback'
spacy en_core_web_md tokenizer (lemma, no stop/punct)

MODULE 3: CONVERSATION (backend/conversation_tracker.py)
- Per-user maps: add_message(msg, intent), get_context_for_llm()

MODULE 4: FRONTEND JS (Frontend/script.js)
- Auth: fetch('/api/auth/guest') token -> localStorage
- Voice: recognition.continuous=false, interimResults=true
- UI: sidebar history, toast notifications, theme toggle

5.4 INTERPRETATION OF RESULTS
- **Hybrid Superiority:** Classical ML (87% acc) + LLM fallback reduces hallucinations 4x vs pure LLM
- **Real-time Viable:** 1.38s avg enables natural conversation flow
- **Scalable:** MongoDB handles 1000s convos, concurrent ThreadPoolExecutor
- **Impact:** 85%+ DEI accuracy → reliable education tool
- **Edge Cases:** Voice WER 91% Chrome (fallback typing), Hindi embeddings drop 5% vs EN

Limitations: API costs, no offline mode. Meets all success criteria.
