### Explanation:
The Inclusivity AI Chatbot follows a **3-tier modular architecture** ensuring separation of concerns, scalability, and maintainability:

**1. Presentation Tier (Frontend)**:
- Responsive single-page app (Bootstrap 5 + Vanilla JS)
- Concurrent message queue (`Promise.all()` for parallel API calls)
- Voice interface (Web Speech API: speech-to-text)
- Real-time UI updates, conversation history sidebar

**2. Application Tier (Flask API Server)**:
- RESTful endpoints with JWT authentication (email/password + Google OAuth)
- Core `/api/chat` endpoint orchestrates hybrid AI pipeline
- Thread-safe concurrent processing (`asyncio` for LLM)
- CORS enabled for local dev (`localhost:8000`, `file://`)

**3. Data Tier (MongoDB)**:
- `users`: `{email, password_hash/auth_provider, conversations[]}`
- `conversations`: `{userId, title, messages[{sender, content, timestamp}]}`
- `dei_dataset`: Loaded from CSV (~10k records for training/reference)

**4. AI Services Layer**:
- `nlp_service.py`: Intent classification (TF-IDF + SGDClassifier)
- `llm_service.py`: Google Gemini 1.5 Flash fallback
- `conversation_tracker.py`: Session context management

This architecture supports **high concurrency** (5+ messages seamless), **security** (JWT HS256), and **extensibility** (microservices-ready).

**Explanation:**
User queries flow through **4 sequential stages** with intelligent branching:

1. **Input Layer**: Text direct, voice via browser-native Web Speech API
2. **Preprocessing**: SpaCy (`en_core_web_md`) lemmatization, stopword removal
3. **Intent Classification**: SGDClassifier predicts intent + confidence via `predict_proba()`
   - вүҘ50% conf вҶ’ Lookup hardcoded DEI response
   - <50% вҶ’ Gemini LLM with conversation context
4. **Persistence**: All exchanges saved to MongoDB with timestamps
5. **Output**: Low-latency (<1.4s avg) streaming response

This **hybrid approach** ensures 87% accuracy for known intents while handling novel queries gracefully.

## 3.3 Algorithms Used

### Algorithm 3.3.1: Intent Classification

**Input**: User query `Q` (string)  
**Output**: Intent label + confidence score  

**Pseudocode**:
```
1. tokens = spacy_tokenizer(Q)  // Lemmatize, remove stopwords/punct
2. V(Q) = TfidfVectorizer.transform([tokens])
3. probs = SGDClassifier.predict_proba(V(Q))
4. max_conf = max(probs[0])
5. intent = argmax(probs[0])
6. IF max_conf >= 0.50:
     RETURN intent
   ELSE:
     RETURN "nlu_fallback"
```

**Implementation** (from `nlp_service.py`):
```python
class IntentClassifier:
    pipeline = Pipeline([
        ('tfidf', TfidfVectorizer(tokenizer=spacy_tokenizer)),
        ('clf', SGDClassifier(loss='log_loss', penalty='l2'))
    ])
    
    def predict(self, text):
        probs = self.pipeline.predict_proba([text])
        max_conf = np.max(probs)
        if max_conf >= 0.50:
            return self.pipeline.classes_[np.argmax(probs)]
        return "nlu_fallback"
```
**Training**: On `data/nlu.yml` (~2000 examples across 10+ DEI intents: `greet`, `ask_bias`, `ask_equity`, etc.)

### Algorithm 3.3.2: Semantic Search (Planned Enhancement)

**Input**: Query embedding `E(Q)`, dataset embeddings `E(D)`  
**Steps**:
1. `sim = cosine_similarity(E(Q), E(D_i))` for all i
2. Sort descending, threshold 0.7
3. Retrieve top-K responses

**Note**: Currently augmented via LLM context; full SentenceTransformers integration pending.

**Input**: Query `Q`, conversation context `C`

def get_generative_response(self, prompt, context):
